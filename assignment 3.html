Sorting is a process that rearange a set of data in some meaningful order. e.g, in increasing order.

Definition: sorted in increasing order

For a given list A={a1,a2,…,an}A = \{ a_1, a_2, \dots, a_n \}A={a1​,a2​,…,an​},
we say AAA is sorted in increasing order if ∀i,1<i≤n,ai−1≤ai\forall i, 1 \lt i \le n, a_{i-1} \le a_i∀i,1<i≤n,ai−1​≤ai​

Sorted data may make our work musch easier than arbitary data.
For a given list A={a1,a2,…,an}A = \{ a_1, a_2, \dots, a_n \}A={a1​,a2​,…,an​},
finding the maximum element in AAA at least cost O(n)O(n)O(n) time.
Since we need to check the value of each elements at least once to guarentee the answer,
our time complexity always not better than O(n)O(n)O(n).
We may design a linear time element as below:

template <typename T>
T& max(const T data[], int n) {
    int ret = 0;
    for (int i = 1; i < n ; ++i) {
        if (data[ret] < data[i])
            ret = i;
    }
    return data[ret];
}

If our data is in increasing order, the only thing we need to do is just return the last element (cost O(1)O(1)O(1)).

template <typename T>
T& max(const T sorted_data[], int n) {
    return sorted_data[n-1];
}

對於 n 個不同元素的排序問題，我們可以將任何比較式排序演算法建成一棵決策樹（decision tree）。

思考方式：
每個比較操作（例如 a[i] < a[j]）都相當於樹中的一個節點，
每個比較結果都有兩種情況（true / false），
每一條從 root 到 leaf 的路徑都對應一種元素的排列結果，
所以這棵樹的 葉節點數量 必須大於等於可能的輸出，也就是 n! 個不同的排列組合。
數學推導：

假設 decision tree 的高度為 h，那麼它最多有 2^h 個葉節點。
因此我們需要：

2h≥n!⇒h≥log2(n!)2^h ≥ n! ⇒ h ≥ log₂(n!)2h≥n!⇒h≥log2​(n!)

根據 Stirling 公式估算：

log₂(n!) ≈ n log₂(n) - n log₂(e) + O(log n)

因此：

h = Ω(n log n)

結論：

任何基於比較的排序法，其最壞情況下的時間複雜度至少為 Ω(n log n)。這是它們的理論下限，無法突破。

Binary search

Let us do a more complex variation of above problem.
For a given list A={a1,a2,…,an}A = \{ a_1, a_2, \dots, a_n \}A={a1​,a2​,…,an​} and a key kkk, asking for finding the elements a∈Aa \in Aa∈A that a=ka = ka=k.

In naive way, we can design an O(n)O(n)O(n) program as below:

template <typename T>
int find(const T data[], int n, const T key) {
    for (int i = 0; i < n ; ++i) {
        if (data[i] == key)
            return i;
    }
    return n; // return terminator to represent not found
}

And again, if we working on a sorted list, we may have a much faster solution.

Suppose we choosing an arbitary element aka_kak​ from an incrasing order list AAA.
We know that any element ai,i<ka_i, i < kai​,i<k will not bigger than aka_kak​,
and any element ai,i>ka_i, i > kai​,i>k will not less than aka_kak​.

Thus, we can design a search progrm as followed:

template <typename T>

int find(const T sorted_data[], int n, const T key) {
    if (n <= 0)
        return 0;
    int k = n/2; // choose arbitary element a_k
    if (sorted_data[k] == key)
        return k;
    else if (sorted_data[k] > key) { // target i < k
        int tmp = find(sorted_data, k, key);
        return (tmp == k)? tmp: n; // if not found, return terminator
    } else { // target i > k => start from k+1
        int tmp = find(sorted_data + k + 1, n - k - 1, key);
        return tmp + k + 1;
    }
}

Above algorithm named binary search which always choose mid element as period and divide data to half and half.
Binary search cost O(log⁡2n)O(\log_2 n)O(log2​n) time, which is much faster than O(n)O(n)O(n).
If we consider the preprocessing cost (O(nlog⁡n)O(n \log n)O(nlogn) for sorting), binary search will beat naive algorithm when we query more than log⁡n\log nlogn times on the same data set.
Empirically, the larger the amount of data, the more often is is reused.
That's why binary search is the first choice of most of database management system.

We can solve unkonwn continuous function by intermediate value theorem and binary search as below:

#include <cmath>

double solve(double (*f)(double), double a, double b, double val, double epi) {
    if ((f(a) - val) * (f(b) - val) > 0)
        return NAN;
    double mid = (a + b) * 0.5;
    if (abs(f(mid) - val) <= epi)

kkk-ary search

For traditional search problem, we can divide data to more than 2 partitions if we want.

Followed pseudo code shows how to work with kkk partitions

#include <vector>
template <typename T>

int find(const std::vector<T> &data, const T key, int k, int lb, int ub) {
    if (ub - lb < k) {
        for (int i = lb; data[i] <= key && i < ub; ++i)
            if (data[i] == key)
                return i;
        return data.size();
    }
    int periods[k - 1];
    for (int i = 1; i < k; ++i) 
        periods[i-1] = lb * i + ub * (k-i) / k;
    if (key < data[periods[0]])
        return find(data, key, k, lb, periods[0]);
    if (key == data[periods[0]])
        return periods[0];
    for (int i = 1; i < k-1; ++i) {
        if (key < data[periods[i]])
            return find(data, key, k, periods[i-1]+1, periods[i]);
        if (key == data[periods[i]])
            return periods[i];
    }
    return find(data, key, k, periods[k-2]+1, ub);
}

int find(const std::vector<T> &data, const T key, int k) {
    return find(data, key, k, 0, data.size());
}
